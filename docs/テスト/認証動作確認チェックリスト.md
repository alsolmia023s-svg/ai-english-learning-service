# Google認証機能 動作確認チェックリスト

## 1. 画面表示・遷移確認
- [ ] **ログインボタンの表示**
  - トップページヘッダーに「Login」ボタンが表示されていること（未ログイン時）。
  - 「Login」ボタンをクリックすると `/auth/signin` に遷移すること。
- [ ] **ログイン画面の表示**
  - `/auth/signin` に「Googleでログイン」ボタンが表示されていること。

## 2. 認証プロセス確認
- [ ] **Googleログイン**
  - 「Googleでログイン」ボタンをクリックすると、Googleの認証画面に遷移すること。
  - アカウント選択後、アプリにリダイレクトされること。
- [ ] **ログイン後のリダイレクト**
  - ログイン成功後、ダッシュボード (`/dashboard`) に遷移すること。

## 3. ログイン後の表示確認
- [ ] **ヘッダー表示**
  - ヘッダーにユーザー名とアバターが表示されていること。
  - アバターをクリックするとメニュー（ダッシュボード、ログアウト）が表示されること。
- [ ] **ダッシュボード表示**
  - `/dashboard` にユーザー情報（名前、メールアドレス、権限）が表示されていること。

## 4. ログアウト確認
- [ ] **ログアウト処理**
  - ヘッダーメニューの「ログアウト」をクリックするとログアウトされ、トップページに遷移すること。
  - ログアウト後、ヘッダーが未ログイン状態（「Login」ボタン表示）に戻ること。

## 5. アクセス制御確認
- [ ] **未認証アクセス制限**
  - ログアウト状態で `/dashboard` に直接アクセスすると、ログイン画面 (`/auth/signin`) にリダイレクトされること。

## 6. データ保存確認
- [ ] **MongoDB保存**
  - 初回ログイン時に、MongoDBの `users` コレクションにユーザー情報が保存されていること。
