# 3. ユーザーニーズ洗い出し

## コアニーズ (Must have)
ユーザーがサービスを利用する上で最も基本的かつ重要な欲求。

1. **正確で自然な英語に直してほしい**
   - 単なる文法チェックだけでなく、ネイティブが使うような自然な表現への書き換えを求めている。
   - ビジネス/アカデミックなど、文脈に合わせたトーン＆マナーの調整が必要。

2. **修正の理由を知りたい**
   - なぜ修正されたのか、元の文のどこが悪かったのかの解説が欲しい。
   - ただ直されるだけでは学習にならず、応用が利かないため。

3. **手軽に素早く使いたい**
   - ログインから結果取得までのステップを最小限にしたい。
   - 待ち時間は数秒以内が望ましい (Claude Haiku 4.5の高速性を活かす)。

## サブニーズ (Should have)
体験の質を高め、継続利用を促す欲求。

4. **過去の学習内容を振り返りたい**
   - 以前添削してもらった内容を履歴として保存し、後で見返したい。
   - 自分のミスの傾向を知りたい。

5. **質問したい**
   - AIの修正に対して「こういう言い方はどう？」と追加で質問したい（MVPではスコープ外の可能性ありだが、ニーズとしては高い）。

## 潜在ニーズ (Could have)
ユーザー自身も明確に言語化していないかもしれないが、あると嬉しい要素。

6. **モチベーションを維持したい**
   - 学習の進捗が可視化されると嬉しい（投稿数、学習日数など）。
   - 毎日の習慣化をサポートしてほしい。

7. **スマホでも使いたい**
   - 移動中や外出先でも手軽にチェックしたい。
